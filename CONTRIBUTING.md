# 贡献指南

感谢你对 Sub2API Response Transformer 的关注。

## 提交前检查

1. 保持修改聚焦，不混入无关格式化或生成文件。
2. 为新增的匹配或代理行为补充测试。
3. 在本地运行：

```bash
go test ./...
go vet ./...
```

4. 不要提交 `config.json`、`config.local.json`、令牌、上游凭据或构建产物。

## Pull Request

请在 PR 描述中说明：

- 解决的问题和预期行为
- 配置或 API 的兼容性影响
- 已执行的验证命令
- 涉及响应转换时的输入和预期输出示例

## 安全问题

请不要在公开 Issue 中披露可被利用的安全问题或真实令牌。请通过仓库维护者的私有联系方式报告，并提供可复现步骤和影响范围。
